CSS Position

The position property in CSS specifies how an element is positioned in the document. This page covers the different position values, their definitions, and examples.

Static

Default value: Elements are positioned according to the normal document flow.

.static {
  position: static;
}
This is a static element.

Relative

Elements are positioned relative to their normal position. Use top, right, bottom, and left to adjust their position. 

.relative {
  position: relative;
  top: 10px;
  left: 20px;
}
This is a relative element.

Absolute

Elements are positioned relative to the nearest positioned ancestor (not static). If no ancestor is positioned, it is relative to the viewport.

.absolute {
  position: absolute;
  top: 10px;
  left: 20px;
}
This is an absolute element.
This is the positioned ancestor.

Fixed

Elements are positioned relative to the viewport and do not move when scrolling.

.fixed {
  position: fixed;
  top: 10px;
  right: 10px;
}

Note: The fixed example appears in the top-right corner of the viewport and may overlay other content â€” look at the top-right of your browser window.

This is a fixed element.

Sticky

Elements toggle between relative and fixed based on the scroll position. Use top, right, bottom, or left to define the sticky threshold. 

.sticky {
  position: sticky;
  top: 0;
}
This is a sticky element.

Important Considerations

  • Stacking Context: Positioned elements can create new stacking contexts, affecting z-index.
  • Viewport Units: Use vh and vw for fixed positioning relative to the viewport.
  • Overflow: Ensure parent containers have appropriate overflow settings to avoid clipping.
  • Sticky Support: Check browser compatibility for position: sticky.
