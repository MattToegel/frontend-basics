Core HTML tags

Short, practical examples of the most common and important HTML tags for valid and accessible pages.

Document skeleton

Every page should start with the HTML5 doctype and a language on the root element. This ensures proper rendering and accessibility.

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>Page title</title>
  </head>
  <body>
    <!-- content -->
  </body>
</html>

Head & metadata

The <head> element contains metadata about the document, such as its title, character encoding, and links to external resources like stylesheets and scripts.

  • <meta charset="utf-8">: Specifies the character encoding for the document.
  • <meta name="viewport" content="width=device-width,initial-scale=1">: Ensures the page is responsive on mobile devices.
  • <link rel="stylesheet" href="styles.css">: Links to an external CSS file for styling.
  • <script src="app.js"></script>: Includes an external JavaScript file.
<meta name="viewport" content="width=device-width,initial-scale=1">
<link rel="stylesheet" href="styles.css">
<script src="app.js"></script>

Text & headings

Use headings (<h1>–<h6>) and paragraphs for readable content. Headings provide structure, while paragraphs group related text.

  • <h1>: Main heading, used once per page.
  • <h2>–<h6>: Subheadings, used to organize content hierarchically.
  • <p>: Defines a paragraph of text.
  • <strong>: Indicates strong importance (bold).
  • <em>: Indicates emphasis (italic).
<h1>Main heading</h1>
<p>A paragraph with <strong>strong</strong> and <em>emphasis</em>.</p>

Links, images & media

Links and images are essential for navigation and visual content. Always include alt attributes for images to improve accessibility.

  • <a>: Creates a hyperlink. Use rel="noopener" for security when opening links in a new tab.
  • <img>: Embeds an image. The alt attribute provides alternative text for screen readers.
  • <figure> and <figcaption>: Group images with captions for better context.
<a href="https://example.com" target="_blank" rel="noopener">External link</a>
<img src="photo.jpg" alt="Person smiling">

<figure>
  <img src="diagram.svg" alt="Flow diagram">
  <figcaption>Diagram caption</figcaption>
</figure>

Lists, tables & code

Lists and tables organize content, while <pre> and <code> are used for displaying code samples.

  • <ul>: Unordered list, for items without a specific order.
  • <ol>: Ordered list, for sequential items.
  • <table>: Displays tabular data. Use <thead>, <tbody>, and <th> for structure.
  • <pre>: Preserves whitespace and formatting for code.
  • <code>: Inline code or code blocks.
<ul>
  <li>First item</li>
</ul>

<table>
  <thead><tr><th>Name</th><th>Age</th></tr></thead>
  <tbody><tr><td>Alex</td><td>30</td></tr></tbody>
</table>

<pre><code>console.log('Hello')</code></pre>

Forms & inputs

Forms collect user input. Use appropriate input types and labels for better validation and accessibility.

  • <form>: Groups form elements.
  • <label>: Associates a label with an input field.
  • <input>: Collects user input. Use type attributes like text, email, or password.
  • <textarea>: Multi-line text input.
  • <select>: Dropdown menu for selecting options.
  • <button>: Submits the form or triggers actions.
<form action="/submit" method="post">
  <label for="email">Email</label>
  <input id="email" name="email" type="email" required>

  <label for="message">Message</label>
  <textarea id="message" name="message"></textarea>

  <label for="country">Country</label>
  <select id="country" name="country">
    <option value="us">United States</option>
  </select>

  <button type="submit">Send</button>
</form>

Semantic structure

Semantic elements provide meaning to the content and improve accessibility. Use them to structure your page logically.

  • <header>: Introductory content or navigation links.
  • <nav>: Navigation links.
  • <main>: Main content of the document.
  • <article>: Self-contained content, like a blog post.
  • <aside>: Related content, like a sidebar.
  • <footer>: Footer content, like copyright or contact info.
<header>...</header>
<nav>...</nav>
<main>
  <article>...</article>
  <aside>...</aside>
</main>
<footer>...</footer>

Inline vs block-level & void elements

HTML elements are categorized as inline, block-level, or void. Understanding these helps with layout and structure.

  • Inline: <a>, <img>, <span>. These do not start on a new line.
  • Block-level: <div>, <p>, <section>. These start on a new line and take up the full width.
  • Void: <img>, <br>, <hr>, <input>. These do not have closing tags.

Accessibility & best practices

Follow these practices to ensure your HTML is accessible and user-friendly:

  • Include alt attributes on images for screen readers.
  • Use label elements for form controls.
  • Set the lang attribute on the <html> element for language-specific content.
  • Validate your HTML with an online validator to catch errors.
  • Use semantic elements for better structure and meaning.

Quick checklist

  1. Start with <!DOCTYPE html>.
  2. Add <meta charset="utf-8"> and a <title>.
  3. Use meaningful tags and attributes such as alt, for, and href.
  4. Keep markup valid and properly nested.
