What is <!DOCTYPE>?

The DOCTYPE is a short instruction at the very top of an HTML file that tells the browser which version of HTML to expect. It helps the browser render the page correctly and avoid "quirks mode."

Why it matters

  • Ensures consistent rendering across browsers.
  • Prevents browsers from switching to legacy "quirks mode".
  • Makes your page compatible with modern HTML features and validators.

HTML5 DOCTYPE (recommended)

HTML5 uses a simple, short DOCTYPE. Place it as the very first line in your file: 

<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="utf-8">
        <title>My page</title>
    </head>
    <body>
        <!-- content -->
    </body>
</html>

Older doctypes

Older HTML versions (HTML4, XHTML) used longer DOCTYPE declarations. You generally only need them if you're maintaining legacy sites. 

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"
    "http://www.w3.org/TR/html4/strict.dtd">

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN"
    "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

Common mistakes

  • Putting the DOCTYPE after comments or whitespace — it must be the first thing.
  • Using the wrong DOCTYPE for the document type — stick to the HTML5 DOCTYPE for new pages.

Quick checklist

  1. Place <!DOCTYPE html> at the very top of the file.
  2. Use <html lang="...")> for accessibility and proper language handling.
  3. Validate with an HTML validator if you need strict compatibility.
